# Project Overview

Static SPA admin dashboard for Triad.

Consumes the Triad API (`/api/v1/admin/*`) for moderation and administration.

Primary stack:

- Vanilla JS (no framework, Preact only if strictly needed)
- Tailwind CSS (CDN, dark theme)
- i18n via JSON locale files
- Mock API for offline development
- Deployed to Cloudflare R2 via GitHub Actions

---

# Architecture

Single-page application with hash-based routing.

- `index.html` → entry point, loads all scripts
- `config.js` → runtime config (API_URL, feature flags), generated by CI
- `js/app.js` → boot, route registration
- `js/router.js` → hash-based SPA router
- `js/pages.js` → page render functions
- `js/api.js` → API client (fetch wrapper, auth token management)
- `js/mock-api.js` → fake data for development
- `js/i18n.js` → translation system
- `locales/` → JSON translation files (en, es)
- `.github/workflows/deploy.yml` → CI/CD to R2

---

# Rules

- Zero build step — open index.html and it works
- Vanilla JS only, no jQuery, no heavy frameworks
- Preact allowed ONLY if vanilla becomes unmanageable
- All text must use i18n keys, never hardcoded strings
- Dark theme always, responsive always
- config.js is gitignored in production (generated by CI)
- No secrets in the frontend — only API_URL and feature flags

---

# i18n

- Locale files in `/locales/{lang}.json`
- Detected from `navigator.language`, stored in localStorage
- User can override via language selector in nav
- All UI text uses `i18n.t("key")` — never raw strings
